## Overview

The event system tracks all significant actions, state changes, and user interactions during a session. Events are stored in a ring buffer and can be polled or attached to tool responses.

## EventQueue Class

The `EventQueue` class manages events for a session using a ring buffer with automatic sequence numbering.

### Constructor Options

<ResponseField name="maxEvents" type="number" default="1000">
  Maximum number of events to retain in the ring buffer
</ResponseField>

### Event Structure

Each event contains:

<ResponseField name="id" type="string">
  Unique event identifier (random base64url string)
</ResponseField>

<ResponseField name="seq" type="number">
  Monotonically increasing sequence number (internal)
</ResponseField>

<ResponseField name="source" type="string">
  Event source: `agent` | `human` | `system`
</ResponseField>

<ResponseField name="type" type="string">
  Event type: `input` | `capture` | `watch-trigger` | `error` | etc.
</ResponseField>

<ResponseField name="tickIndex" type="number | null">
  Game tick index when the event occurred (null if not applicable)
</ResponseField>

<ResponseField name="time" type="string">
  ISO 8601 timestamp
</ResponseField>

<ResponseField name="summary" type="string">
  Human-readable event summary
</ResponseField>

<ResponseField name="data" type="object">
  Optional structured event data
</ResponseField>

<ResponseField name="resourceUris" type="string[]">
  Array of resource URIs associated with this event
</ResponseField>

## Event Sources

### Agent Events

Generated by MCP tool calls:

```javascript
session.events.add({
  source: 'agent',
  type: 'input',
  summary: 'action:pause x1',
  tickIndex: 120
});
```

**Common agent event types:**
- `input`: Keyboard/mouse actions
- `capture`: Screenshot or sequence capture

### Human Events

Generated by spectator UI interactions:

```javascript
session.events.add({
  source: 'human',
  type: 'input',
  summary: 'key:press:Space',
  tickIndex: 125
});
```

<CodeGroup>

```javascript Key Event
{
  source: 'human',
  type: 'input',
  summary: 'key:down:ArrowLeft'
}
```

```javascript Click Event
{
  source: 'human',
  type: 'input',
  summary: 'click:0.45,0.62'
}
```

</CodeGroup>

### System Events

Generated by watches, errors, and internal processes:

```javascript
session.events.add({
  source: 'system',
  type: 'watch-trigger',
  summary: 'watch:abc123 summary',
  tickIndex: 200,
  data: {
    lemmingsSummary: { ... },
    statePointers: { '/game/victory/outCount': 5 }
  }
});
```

**Common system event types:**
- `watch-trigger`: Watch condition met
- `error`: Page errors, console errors, capture failures
- `capture`: Watch-triggered captures

## Core Methods

### add()

Add a new event to the queue.

```javascript
const entry = events.add({
  source: 'agent',
  type: 'input',
  summary: 'action:selectSkillBuilder x1',
  data: { action: 'selectSkillBuilder' },
  resourceUris: [],
  tickIndex: 150
});
```

**Behavior:**
- Assigns a unique `id` and incremental `seq`
- Adds current ISO timestamp
- If `source === 'human'` and `summary` is non-empty, appends to `humanSummaryParts`
- Evicts oldest event if `maxEvents` limit is exceeded

**Returns:** The complete event entry with `id` and `seq`

### drain()

Retrieve all events since a cursor.

```javascript
const envelope = events.drain(afterSeq, {
  updateCursor: true,
  includeHumanSummary: true
});
```

<ResponseField name="after" type="string | number">
  Cursor (sequence number) to read after. Defaults to `lastDelivered`
</ResponseField>

<ResponseField name="updateCursor" type="boolean" default="true">
  Whether to update the `lastDelivered` cursor
</ResponseField>

<ResponseField name="includeHumanSummary" type="boolean" default="true">
  Whether to include the aggregated human summary
</ResponseField>

**Returns:**

```javascript
{
  cursor: '152',  // Sequence number of last event in this batch
  events: [
    { id, source, type, tickIndex, time, summary, data, resourceUris },
    ...
  ],
  humanSummary: 'key:press:Space; click:0.45,0.62'  // Joined summaries
}
```

Returns `null` if no new events.

<Note>
The `humanSummary` field aggregates all human-sourced event summaries since the last drain, then clears the buffer. This provides a concise "what the human did" string for agent awareness.
</Note>

## Events Mode

Configure event verbosity when creating a session:

```javascript
{
  events: {
    mode: 'minimal'  // 'none' | 'minimal' | 'full'
  }
}
```

### None

- No event envelopes attached to tool responses
- Use `events.poll` to explicitly fetch events

### Minimal (Default)

- Includes only non-agent events (human + system)
- Trims fields to: `source`, `type`, `tickIndex`, `summary`, `resourceUris`
- Omits `id`, `time`, and `data`
- Empty events array is omitted from response

<CodeGroup>

```javascript Minimal Event
{
  source: 'human',
  type: 'input',
  tickIndex: 125,
  summary: 'key:press:Space',
  resourceUris: []
}
```

```javascript With Resources
{
  source: 'system',
  type: 'capture',
  tickIndex: 200,
  summary: 'watch:abc123 capture',
  resourceUris: ['lemmings://sessions/xyz/resources/def456']
}
```

</CodeGroup>

### Full

- Includes all events (agent, human, system)
- All fields preserved: `id`, `seq`, `source`, `type`, `tickIndex`, `time`, `summary`, `data`, `resourceUris`

## Attaching Events to Tool Responses

The `attachEvents()` helper automatically adds event envelopes to tool responses:

```javascript
const response = attachEvents(session, {
  ok: true,
  tickIndex: 150,
  // ... tool-specific fields
});
```

**Behavior based on events mode:**

<CodeGroup>

```javascript None
// No events field added
{
  ok: true,
  tickIndex: 150
}
```

```javascript Minimal
{
  ok: true,
  tickIndex: 150,
  events: {
    cursor: '152',
    events: [
      { source: 'human', type: 'input', summary: '...' }
    ],
    humanSummary: 'key:press:Space'
  }
}
```

```javascript Full
{
  ok: true,
  tickIndex: 150,
  events: {
    cursor: '152',
    events: [
      { id, seq, source, type, tickIndex, time, summary, data, resourceUris }
    ],
    humanSummary: 'key:press:Space'
  }
}
```

</CodeGroup>

## Polling Events

Use the `events.poll` tool to explicitly fetch events:

```javascript
const result = await tool('events_poll', {
  sessionId: 'xyz',
  after: '150'  // Optional cursor
});
```

**Returns:**

```javascript
{
  cursor: '158',
  events: [ ... ],
  humanSummary: 'Human pressed Space (pause) and ArrowRight x3'
}
```

Returns empty/null envelope if no events since cursor.

## Ring Buffer Behavior

The event queue is a **fixed-size ring buffer**:

- Maximum size set by `maxEvents` (default 1000)
- Oldest events are discarded when limit is reached
- Sequence numbers continue to increment (monotonic)

<Note>
If you poll infrequently and the buffer wraps, you may miss events. Poll regularly or increase `maxEvents` for high-activity sessions.
</Note>

## Human Summary Aggregation

The `humanSummaryParts` array accumulates summaries from human events:

```javascript
// Human presses Space, then ArrowRight twice
events.add({ source: 'human', summary: 'key:press:Space' });
events.add({ source: 'human', summary: 'key:press:ArrowRight' });
events.add({ source: 'human', summary: 'key:press:ArrowRight' });

// Next drain() returns:
{
  humanSummary: 'key:press:Space; key:press:ArrowRight; key:press:ArrowRight'
}

// humanSummaryParts is cleared after drain
```

<Info>
This provides a concise "what changed" string even when the MCP host doesn't consume streaming notifications. The agent receives a summary of human actions on the next tool call.
</Info>

## Best Practices

<Note>
**Use minimal mode by default**: Reduces noise and token usage. Agent events are echoed back in the response already.
</Note>

<Note>
**Poll before long-running operations**: Drain events before starting multi-step sequences to capture any human interventions.
</Note>

<Note>
**Watch the cursor**: Store the cursor from each drain and pass it to the next poll to avoid duplicate events.
</Note>

<Note>
**Clear sessions to reset events**: Events are per-session. Close and recreate the session to start fresh.
</Note>
