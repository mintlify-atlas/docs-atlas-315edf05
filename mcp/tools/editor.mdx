## editor.apply

Apply editor mutations through the E2E harness. Supports atomic operations, dry runs, validation, and state export.

### Parameters

<ParamField path="sessionId" type="string" required>
  Session identifier.
</ParamField>

<ParamField path="ops" type="array<object>" optional>
  Array of editor operations to apply.
  <ParamField path="ops[].opId" type="string" optional>
    Optional operation ID for tracking.
  </ParamField>
  <ParamField path="ops[].type" type="string" required>
    Operation type (e.g., `level.new`, `entry.add`, `entry.update`, `entry.delete`, `level.export`).
  </ParamField>
  <ParamField path="ops[].args" type="any" optional>
    Operation-specific arguments.
  </ParamField>
</ParamField>

<ParamField path="atomic" type="boolean" optional>
  If true, all operations succeed or all fail together.
</ParamField>

<ParamField path="dryRun" type="boolean" optional>
  If true, validate operations but don't apply changes.
</ParamField>

<ParamField path="history" type="object" optional>
  History tracking options.
  <ParamField path="history.label" type="string" optional>
    Label for the history entry.
  </ParamField>
  <ParamField path="history.record" type="boolean" optional>
    Whether to record this operation in undo history.
  </ParamField>
</ParamField>

<ParamField path="preview" type="object" optional>
  Preview refresh options.
  <ParamField path="preview.refresh" type="boolean" optional>
    Refresh the preview after operations.
  </ParamField>
  <ParamField path="preview.label" type="string" optional>
    Label for the preview refresh.
  </ParamField>
  <ParamField path="preview.preserveViewport" type="boolean" optional>
    Preserve viewport position during refresh.
  </ParamField>
</ParamField>

<ParamField path="validate" type="object" optional>
  Validation options.
  <ParamField path="validate.run" type="boolean" optional>
    Run validation after applying operations.
  </ParamField>
  <ParamField path="validate.autoFix" type="enum<'none' | 'safe' | 'aggressive'>" optional>
    Automatically fix validation issues.
  </ParamField>
</ParamField>

<ParamField path="returnState" type="enum<'none' | 'editor' | 'full'>" optional>
  What state to return:
  - `none`: No state returned
  - `editor`: Editor state only
  - `full`: Complete E2E state
</ParamField>

### Response

<ResponseField name="ok" type="boolean" required>
  Overall operation success.
</ResponseField>

<ResponseField name="results" type="array<object>" required>
  Results for each operation.
  <ResponseField name="results[].opId" type="string" optional>
    Operation ID if provided.
  </ResponseField>
  <ResponseField name="results[].ok" type="boolean" required>
    Operation success status.
  </ResponseField>
  <ResponseField name="results[].error" type="string" optional>
    Error message if operation failed.
  </ResponseField>
  <ResponseField name="results[].result" type="any" optional>
    Operation-specific result data.
  </ResponseField>
</ResponseField>

<ResponseField name="state" type="object" optional>
  Editor or full state (if `returnState` is not `none`).
</ResponseField>

<ResponseField name="resources" type="array<object>" required>
  Resources generated by operations (e.g., exported level files).
  <ResponseField name="resources[].uri" type="string" required>
    Resource URI.
  </ResponseField>
  <ResponseField name="resources[].mimeType" type="string" required>
    MIME type.
  </ResponseField>
  <ResponseField name="resources[].name" type="string | null" required>
    Resource name.
  </ResponseField>
  <ResponseField name="resources[].sizeBytes" type="integer" required>
    Size in bytes.
  </ResponseField>
  <ResponseField name="resources[].meta" type="object" optional>
    Resource metadata.
  </ResponseField>
</ResponseField>

<ResponseField name="events" type="object" optional>
  Event envelope (if events mode is not 'none').
</ResponseField>

### Example

<CodeGroup>
```json Create and Export Level
{
  "sessionId": "abc123def456",
  "ops": [
    {
      "opId": "create",
      "type": "level.new",
      "args": {
        "header": {
          "TITLE": "My Level",
          "STYLE": "dirt"
        }
      }
    },
    {
      "opId": "add-terrain",
      "type": "entry.add",
      "args": {
        "kind": "terrain",
        "props": {
          "PIECE": 1,
          "X": 120,
          "Y": 80
        }
      }
    },
    {
      "opId": "export",
      "type": "level.export",
      "args": {
        "format": "nxlv",
        "filename": "my-level.nxlv"
      }
    }
  ],
  "atomic": true,
  "preview": {
    "refresh": true
  },
  "returnState": "editor"
}
```

```json Response

      "result": { "created": true }

        "header": { "TITLE": "My Level", "STYLE": "dirt" },
```

```json Dry Run Validation
{
  "sessionId": "abc123def456",
  "ops": [
    {
      "type": "entry.update",
      "args": {
        "kind": "terrain",
        "index": 0,
        "props": { "X": -100 }
      }
    }
  ],
  "dryRun": true,
  "validate": {
    "run": true
  }
}
```

```json Response
{
  "ok": true,
  "results": [
    {
      "ok": true,
      "result": { "dryRun": true, "wouldApply": true }
    }
  ],
  "state": null,
  "resources": []
}
```
</CodeGroup>

<Note>
  Common operation types:
  - `level.new`: Create a new level
  - `entry.add`: Add terrain, gadget, or steel entry
  - `entry.update`: Update entry properties
  - `entry.delete`: Delete an entry
  - `entry.move`: Move entry to new index
  - `level.export`: Export level to file (nxlv format)
  - `level.import`: Import level from text
  - `header.set`: Set header field value
</Note>

<Warning>
  Editor operations require the session to be in editor mode. Use `?e2e=1&editor=1` when creating the session, or navigate to the editor in the browser.
</Warning>

<Note>
  For detailed editor state structure, see the [e2e-editor-state.md](https://github.com/doublemover/LemmingsJS-MIDI/blob/main/docs/e2e-editor-state.md) file in the source repository.
</Note>
