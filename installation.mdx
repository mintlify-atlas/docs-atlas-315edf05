## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Open your terminal and clone the repository:
    
    ```bash
    git clone https://github.com/doublemover/LemmingsJS-MIDI
    cd LemmingsJS-MIDI
    ```
    
    This creates a local copy of the entire project including all game assets, levels, and tools.
  </Step>
  
  <Step title="Install Dependencies">
    Install all required Node.js packages:
    
    ```bash
    npm install
    ```
    
    This installs development dependencies including:
    - `http-server` for local hosting
    - `mocha` and `chai` for testing
    - `playwright` for end-to-end tests
    - `eslint` for code quality
    - `c8` for code coverage
  </Step>
  
  <Step title="Start the Development Server">
    Launch the local HTTP server:
    
    ```bash
    npm start
    ```
    
    The server starts on port 8080 with CORS enabled.
  </Step>
  
  <Step title="Open in Browser">
    Navigate to:
    
    - **Game**: [https://localhost:8080](https://localhost:8080)
    - **Editor**: [https://localhost:8080/editor.html](https://localhost:8080/editor.html)
    
    <Note>
      The URLs use `https` when running with the HTTPS server (see below). Use `http://localhost:8080` with the standard `npm start` command.
    </Note>
  </Step>
</Steps>

## HTTPS Server (For MCP)

If you need HTTPS for MCP integration or WebMIDI testing:

<Steps>
  <Step title="Generate SSL Certificates">
    Create a `certs/` directory and generate local SSL certificates:
    
    ```bash
    mkdir -p certs
    # Use mkcert or your preferred SSL certificate tool
    mkcert -install
    mkcert -key-file certs/localhost-key.pem -cert-file certs/localhost.pem localhost
    ```
  </Step>
  
  <Step title="Start HTTPS Server">
    Run the HTTPS-enabled server:
    
    ```bash
    npm run start-https
    ```
    
    Access the game at [https://localhost:8080](https://localhost:8080)
  </Step>
</Steps>

<Warning>
  The HTTPS server is required for MCP integration and some browser security features. Standard HTTP is sufficient for offline gameplay and development.
</Warning>

## Configuration Files

Customize your local installation:

### keybindings.json

Modify keyboard controls to your preference. The file maps actions to key codes:

```json
{
  "pause": ["Space"],
  "speedUp": ["Equal", "NumpadAdd"],
  "speedDown": ["Minus", "NumpadSubtract"],
  "stepForward": ["BracketRight"],
  "stepBackward": ["BracketLeft"]
  // ... more bindings
}
```

### midi-mapping.json

Configure default MIDI routing, input mapping, and sequencer parameters. This file provides defaults that are loaded on first run or when values are missing from localStorage.

### config.json

Game-wide settings including level pack paths, asset directories, and engine parameters.

## Progressive Web App

Install LemmingsJS-MIDI as a standalone app:

<Steps>
  <Step title="Access via HTTPS">
    PWA installation requires HTTPS. Use `npm run start-https` or access the online version.
  </Step>

    Your browser will show an install prompt. Click it or:

</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="npm install fails">
    Ensure you're using Node.js 20 or higher:
    
    ```bash
    node --version  # Should show v20.x.x or higher
    ```
    
    Try clearing npm cache:
    
    ```bash
    npm cache clean --force
    npm install
    ```
  </Accordion>
  
  <Accordion title="Port 8080 already in use">
    Either stop the process using port 8080, or modify the `package.json` scripts to use a different port:
    
    ```json
    "start": "http-server --cors=* -p 3000"
    ```
  </Accordion>
  
  <Accordion title="Tests failing">
    Some tests may require a display server. For headless environments, use Xvfb:
    
    ```bash
    xvfb-run npm test
    ```
    
    Ensure Playwright browsers are installed:
    
    ```bash
    npm run test-e2e:install
    ```
  </Accordion>
  
  <Accordion title="HTTPS certificate errors">
    For local SSL certificates, install `mkcert`:
    
    ```bash
    # macOS
    brew install mkcert
    
    # Windows (Chocolatey)
    choco install mkcert
    
    # Linux
    # See https://github.com/FiloSottile/mkcert
    ```
    
    Then generate certificates as shown in the HTTPS setup step.
  </Accordion>
</AccordionGroup>
